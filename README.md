# bidking

bidking 是一个多人竞拍游戏原型，核心体验围绕信息差、心理博弈、出价决策和藏品收集展开。

这个项目采用个人 + AI 协作开发方式推进：个人负责玩法判断、产品取舍、Unity 场景调试和最终验收，AI 辅助完成代码实现、文档整理、问题排查和工程维护。

## 技术栈

- 客户端：Unity + C#
- 服务端：Go
- 实时通信：WebSocket
- 数据库：PostgreSQL 或 MySQL，后续确定
- 缓存：Redis
- 特效：核心玩法闭环稳定后再补充

## 仓库结构

```text
client/     Unity 客户端工程
server/     Go 游戏服务端
shared/     前后端共享的协议和玩法配置
docs/       设计、架构和开发计划文档
infra/      本地开发基础设施模板
scripts/    开发辅助脚本
```

## 开发原则

- 服务端负责竞拍判定、房间状态和结算结果。
- Unity 客户端负责画面展示、玩家输入和本地反馈。
- 共享配置和协议先放在 `shared`，避免前后端规则不一致。
- 特效和表现后置，优先保证竞拍闭环可玩、稳定、可测试。

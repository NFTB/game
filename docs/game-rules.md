# bidking 游戏规则

## 0. 规则定位

bidking 当前参考《竞拍之王》这类多人暗拍信息博弈游戏的核心结构：玩家围绕未知价值的藏品进行判断、试探和出价，通过公共线索、个人道具线索、对手金币信息和风险偏好形成心理博弈。

当前版本的核心体验是：

```text
看仓库 -> 读线索 -> 判断最低价值 -> 暗拍出价 -> 揭示结果 -> 获得藏品 -> 计算收益
```

第一阶段重点不是完整商业化系统，而是验证三个问题：

- 玩家是否能根据有限信息推断仓库价值。
- 暗拍出价是否有紧张感和博弈感。
- 结算时逐件揭示藏品价值是否有反馈和爽感。

## 0.1 当前规则配置入口

当前规则由文档和 `shared/config` 下的配置共同表达：

| 文件 | 作用 |
|------|------|
| `shared/config/game_rules.json` | 全局规则，如人数、轮数、暗拍、仓库、线索、道具限制、NPC 模型 |
| `shared/config/venues.json` | 场所配置，如入场费、每轮藏品数量范围、长期稀有度比例 |
| `shared/config/maps.json` | 地图配置，如公共线索池和每轮公共线索数 |
| `shared/config/collectibles.json` | 藏品配置，如类型、稀有度、尺寸、轮廓、真实价值、卖出价 |
| `shared/config/items.json` | 道具配置，如道具类型、价格、效果和可见性 |
| `shared/config/roles.json` | 角色系统预留 |

文档用于说明玩法语义，配置用于后续客户端和服务端读取。若两者冲突，以本文档为规则口径，再同步修正配置。

## 1. 基础设定

### 玩家

- 每局人数：2 到 4 人。
- MVP 身份：游客。
- 金币：个人持久资产，跨对局保留，初始 1000 万。
- 当前持有金币决定玩家本轮最高可出价。

### 对局

- 每局 5 轮。
- 每轮展示一个仓库，所有玩家共同暗拍这个仓库。
- 每轮获胜者获得仓库内所有藏品。
- 每轮开始先扣除场所入场费。
- 每轮限时出价 30 秒。

## 2. 场所

场所决定本局的入场费、藏品数量范围和长期内容期望。

场所不再定义仓库网格。仓库网格是全局理论空间，当前按 `200 x 10` 处理，用于摆放和展示藏品，不要求填满。

MVP 场所：

- 地下拍卖所。
- 每轮入场费：10 万。
- 每轮藏品数量范围：30 到 60。
- 藏品类型随机，长期整体期望平均分布。
- 藏品稀有度在单局内随机波动，整个游戏长期期望为 `N:R:SR:SSR = 4:3:2:1`。

### 单局游戏期望和整个游戏期望

单局游戏期望是指某一局实际生成出来的内容倾向。它允许波动，例如某一局可能 SR 较多，另一局可能 N 较多。

整个游戏期望是指大量对局统计后的长期均值。地下拍卖所当前的长期稀有度期望为 `4:3:2:1`，类型长期期望为平均分布。

## 3. 地图

每局游戏在选定场所后随机选取一张地图。地图决定公共线索策略。

- 地图线索发给所有玩家。
- 地图线索属于公共信息。
- MVP 先做 1 张地图。
- 玩家使用道具获得的线索只给使用者本人。

### 地图属性

| 属性 | 说明 |
|------|------|
| 名称 | 地图名 |
| 线索池 | 该地图可能提供的线索类型 |
| 每轮公共线索数 | 每轮向所有玩家公开几条地图线索 |

## 4. 仓库和藏品

### 每轮拍卖对象

每轮拍卖对象是一个仓库，而不是单个藏品。

获胜者获得该仓库内所有藏品。未获胜玩家不获得本轮仓库内藏品。

### 仓库生成

- 仓库理论网格：`200 x 10`。
- 每轮藏品数量由场所决定，MVP 为 30 到 60。
- 仓库不要求填满。
- 总价值不单独设定，由藏品类型和稀有度共同决定。
- 藏品类型随机，长期期望平均分布。
- 藏品稀有度随机，长期期望由场所配置决定。

### 摆放规则

- 藏品按格子尺寸摆入仓库。
- 从上到下不能有间隙。
- 最后可以出现凸出物品。
- 凸出物品上方不会再堆叠其他物品。
- 从左到右的列高度是非严格单调递减，也就是左侧列高度大于或等于右侧列高度。

### 藏品属性

| 属性 | 说明 |
|------|------|
| 名称 | 藏品名 |
| 图片 | 藏品外观 |
| 真实价值 | 固定值，用于出售和收益计算 |
| 卖出价 | MVP 等于真实价值 |
| 尺寸 | 占仓库格子数，如 `1x1`、`1x2`、`2x1`、`2x2` |
| 轮廓 | MVP 可由尺寸派生 |
| 稀有度 | `N`、`R`、`SR`、`SSR` |
| 类型 | 瓷器、书画、玉器、青铜器、杂项 |

## 5. 线索和估价

### 线索来源

1. 地图每轮发放公共线索，所有玩家都能看到。
2. 玩家每轮最多使用一个道具，道具产生的线索只给使用者。

### 线索类型

MVP 线索类型：

- 轮廓。
- 稀有度。
- 类型。

### 最低估价

玩家看到的是基于已知信息推导出的最低估价。

- 没有任何有效信息时，最低估价为 `0`。
- 获得轮廓后，按匹配该轮廓的所有可能藏品计算最低真实价值。
- 获得稀有度后，按匹配该稀有度的所有可能藏品计算最低真实价值。
- 获得类型后，按匹配该类型的所有可能藏品计算最低真实价值。
- 同时获得多个线索时，取所有线索交集中的最低真实价值。
- 如果线索对应的是仓库信息，则按仓库内可确认藏品的最低估价合计展示。

## 6. 出价规则

### 出价方式

暗拍：所有玩家同时出价，出价互相不可见。

### 出价约束

- 最小出价：1 金币。
- 不能超过当前持有金币。
- 可选择不出价，视为放弃本次竞拍。
- 每轮获胜后立即扣除出价金币。

## 7. 成交和展示规则

### 成交

- 每轮最高出价者获得整个仓库。
- 获胜者获得仓库内所有藏品。
- 获胜者立即扣除本轮出价。

### 提前拿下阈值

每轮有不同的提前拿下阈值。阈值是一种信息揭示和胜利表现规则。

- 第 1 轮：最高出价 >= 第二名出价 `2.0` 倍。
- 第 2 轮：最高出价 >= 第二名出价 `1.6` 倍。
- 第 3 轮：最高出价 >= 第二名出价 `1.3` 倍。
- 第 4 轮：最高出价 >= 第二名出价 `1.1` 倍。
- 第 5 轮：无倍数要求。

如果有人提前拿下：

- 展示所有玩家的具体出价。
- 进入结算展示。

如果无人提前拿下：

- 展示出价排名。
- 不展示所有人的具体出价。
- 进入结算展示。

### 结算展示

每轮成交后进入结算界面：

- 一个一个展示仓库内藏品。
- 展示每件藏品的真实价值。
- 展示获胜玩家当前收益变化。
- 藏品进入获胜玩家个人仓库。
- 藏品不会自动出售。

## 8. 平局和流拍

如果出现并列最高价：

- 只有并列最高者可以参与加拍。
- 加拍仍然是暗拍。
- 加拍出价必须高于上一轮自己的最高出价。
- 最多加拍 10 轮。
- 仍未决出胜者时，本轮流拍。

流拍后：

- 本轮仓库作废。
- 没有玩家获得仓库内藏品。
- 不扣除本轮出价金币。
- 已经消耗的入场费和道具不返还。

## 9. 道具

### 道具配置

- 道具在开局前由玩家配置。
- 每位玩家最多配置 5 个道具。
- 道具来自玩家个人库存。
- 使用后消耗库存。
- 每轮最多使用 1 个道具。

### 道具结算

- 道具效果只影响使用者本人看到的信息，除非道具本身明确写明影响对手。
- MVP 道具结果不需要全局先后顺序。
- 每位玩家看到自己的道具结果。
- 道具效果由服务端判定，客户端只展示结果。

### MVP 道具

| 道具 | 稀有度 | 效果 |
|------|------|------|
| 放大镜 | N | 随机获取一条本轮仓库相关线索 |
| 轮廓扫描 | R | 揭示本轮所有 N 稀有度藏品的轮廓 |
| 窥探 | N | 查看一名对手的剩余金币 |
| 窃听 | R | 查看一名对手本轮相关出价信息 |

## 10. 角色

角色系统预留，不进入第一阶段核心实现。

后续角色需要定义：

- 角色 ID。
- 角色名称。
- 主动技能或被动技能。
- 技能触发时机。
- 技能是否影响线索、估价、出价、道具或结算。

MVP 代码和配置中只保留角色扩展入口。

## 11. 卖出和资产

- 拍卖结束后，藏品归入玩家个人仓库。
- 藏品不自动出售。
- 玩家可在对局结束后手动选择是否出售。
- MVP 卖出价等于真实价值。
- 金币和藏品跨对局保留。

## 12. 胜负规则

每局结束后计算本局收益。

开局资产：

```text
开局资产 = 开局金币 + 开局持有藏品真实价值总和
```

结算资产：

```text
结算资产 = 当前金币 + 当前持有藏品真实价值总和
```

本局净收益：

```text
本局净收益 = 结算资产 - 开局资产
```

本局净收益最高者获胜。

## 13. NPC 出价模型

第一阶段单机 Demo 需要假玩家。

NPC 出价受线索准确度影响：

- NPC 和玩家一样能看到地图公共线索。
- NPC 可基于线索估算仓库最低价值。
- 线索越明确，NPC 估价越接近仓库真实价值。
- 线索越少，NPC 出价越保守。
- 不同 NPC 可以有不同风险偏好：保守、均衡、激进。

基础公式预留：

```text
NPC估值 = 可见最低估价 + 不确定性补偿
NPC出价 = NPC估值 * 风险系数
```

## 14. 回合流程

```text
选择场所
-> 随机地图
-> 第 N 轮开始，扣除本轮入场费
-> 生成本轮仓库
-> 发放地图公共线索
-> 玩家可使用至多 1 个道具
-> 30 秒暗拍
-> 计算最高价和平局
-> 判断是否提前拿下
-> 展示出价信息或排名
-> 逐件展示藏品和收益
-> 藏品进入获胜者个人仓库
-> 下一轮
```

5 轮结束后：

```text
计算本局净收益
-> 展示排名
-> 玩家可手动出售藏品
-> 返回大厅
```

## 15. 第一阶段实现重点

第一阶段只做本地单机 Demo，不依赖数据库和真实后端。

必须实现：

- 读取或内置基础规则。
- 生成 30 到 60 件藏品组成的仓库。
- 按 `200 x 10` 理论网格进行仓库摆放。
- 每轮生成公共地图线索。
- 玩家每轮最多使用一个道具。
- 玩家和 NPC 同时暗拍。
- 最高价获得整个仓库。
- 支持并列最高价加拍。
- 支持流拍。
- 每轮逐件展示藏品真实价值和收益变化。
- 5 轮结束后计算本局净收益和排名。

暂不实现：

- 注册登录。
- 数据库持久化。
- 真实多人联机。
- 付费系统。
- 好友、聊天、公会、赛季。
- 完整角色技能。
- 复杂特效。

## 16. 后续扩展方向

后续可在当前规则上扩展：

- 更多场所，不同场所拥有不同入场费、藏品数量范围和长期稀有度期望。
- 更多地图，不同地图提供不同公共线索策略。
- 更多道具，包括干扰类和经济类。
- 角色技能，例如影响线索、估价、出价上限、道具效果或结算收益。
- 藏品合成、交易所和收藏柜展示。
- 联机房间、断线重连和服务端权威判定。

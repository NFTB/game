# bidking 交互设计稿

## 概述

PC 优先，横屏布局。分辨率基准 1920×1080。

核心交互原则：
- 左玩家中信息右仓库：对局中左侧固定展示玩家状态和轮次记录，中间展示公共/个人信息与操作，右侧展示仓库和估价。
- 信息分层：公共信息、个人信息、操作区明确分离。
- 单屏对局优先：出价阶段尽量不切换大场景，出价、道具、百科都以弹层或局部面板完成。
- 状态驱动：第一阶段由客户端本地 `GameState` 驱动；后续联机版本替换为服务端状态快照。
- 反馈即时：按钮按下有反馈，倒计时清晰可见，结果逐条揭示，并允许加速或跳过动画。

---

## 0. 视觉风格参考

整体可以参考《竞拍之王》同类界面的布局气质，但不复用原始素材、图标、角色和品牌元素。bidking 的目标是形成自己的美术资产和 UI 语言。

视觉方向：
- 全屏场景背景：场所背景铺满屏幕，例如地下拍卖所、废弃仓库、沉船资料室等；UI 以半透明深色玻璃面板覆盖在场景上。
- HUD 轻量悬浮：顶部居中显示当前轮次，右上角显示金币、设置、关闭等系统按钮，避免大面积实体边框。
- 左侧玩家栏：纵向显示玩家头像、编号、昵称、当前金币和 1~5 轮记录槽，每个槽记录本轮道具和排名。
- 中置信息流：公共线索、NPC 发言、个人道具结果以卡片流展示；每条卡片包含来源头像或图标、标题、说明文本。
- 右侧战利品仓库：右侧固定一个"战利品"面板，显示仓库网格、藏品轮廓、线索高亮和最低估价。
- 底部操作区：底部中央放主要出价按钮或已出价状态，旁边放道具入口、放弃按钮和倒计时。
- 半透明但可读：背景图可以有氛围，但信息区必须加遮罩，保证中文、数字和按钮在暗/亮背景上都能清楚读。

颜色和状态：
- 未知藏品：深色半透明块 + 白色或灰色边框。
- 被线索命中的藏品：使用蓝色、紫色、黄色等发光边框区分线索来源或稀有度。
- 可操作按钮：主按钮用高亮色，次要操作用低饱和图标按钮。
- 负收益用红色，正收益用绿色，估价和金币用白色或银色。

弹层参考：
- 藏品百科：以大半屏半透明弹层覆盖中左区域，左侧筛选品质/类型/轮廓，中间显示藏品卡片列表，右侧仍保留战利品仓库供对照。
- 结算揭示：背景整体压暗，左侧显示获胜者、成交价、战利品价格和利润，右侧显示仓库从轮廓到真实藏品的揭示过程。

## 3. 出价阶段（对局主界面）

出价阶段采用全屏场景背景，界面上叠加半透明 HUD。左侧是玩家与轮次记录，中间是公共/个人信息流，右侧是战利品仓库和估价，底部是出价操作。

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                         第 2/5 轮                         金币 975万  ⚙  × │
│                                                                              │
│ 玩家记录                  信息流                              战利品          │
│ ┌──────────────┐          ┌──────────────────────────┐      ┌──────────────┐ │
│ │ 1 你          │          │ 公共线索                  │      │ 仓库网格      │ │
│ │ 金币 975万    │          │ 探险家资料库              │      │ ┌┬┬┬┬┬┬┬┬┬┐ │ │
│ │ R1 道具 排名2 │          │ 每件藏品平均占格约 2.59   │      │ ││▓▓││▓▓▓│ │ │
│ │ R2 -    -    │          └──────────────────────────┘      │ ││││││▓▓▓│ │ │
│ └──────────────┘          ┌──────────────────────────┐      │ │▓│││││││ │ │
│ ┌──────────────┐          │ 个人线索                  │      │ └┴┴┴┴┴┴┴┴┴┘ │ │
│ │ 2 NPC-A       │          │ 轮廓扫描：显示 5 件轮廓   │      │              │ │

│ │ R2 ?    -    │          ┌──────────────────────────┐      │ 1,200,000    │ │
│ └──────────────┘          │ 窥探结果                  │      │ 已知 5 件     │ │
│ ┌──────────────┐          │ NPC-C 剩余金币：910万     │      │ 未知 25~55 件 │ │
│ │ 3 NPC-B       │          └──────────────────────────┘      │ 命中格高亮    │ │
│ │ 道具 ? 排名2  │                                            └──────────────┘ │
│ └──────────────┘                                                              │
│ ┌──────────────┐                     30s                                      │
│ │ 4 NPC-C       │       [道具]             [ 出价 ]             [放弃]         │
│ │ 道具 ? 排名1  │       已出价时显示：[已出价：650,001]                       │
│ └──────────────┘                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

布局：
- 背景层：当前场所场景图全屏铺底，整体加暗色遮罩，避免信息和背景打架。
- 顶部 HUD：居中显示轮次，右上角显示金币、设置、退出等系统入口。
- 左侧 22%：4 个玩家卡片纵向排列，每个玩家卡片内记录 1~5 轮道具和排名。
- 中间 38%：线索和事件信息流，按时间顺序展示公共线索、个人线索、NPC 行为提示和道具结果。
- 右侧 34%：战利品仓库面板，展示仓库网格、最低估价、已知/未知藏品概览和线索命中位置。
- 底部 10%：出价操作条，主按钮位于底部中间，道具入口在左，放弃或取消在右。

仓库展示：
- 200×10 是理论网格，不直接按原始比例完整铺满屏幕。
- 主视图显示缩放后的仓库轮廓，用颜色块或剪影表示已知尺寸和未知藏品。
- 当仓库内容过高时，右侧支持纵向滚动；顶部或侧边提供缩略总览，标记当前可见区域。
- 玩家第一眼需要看清藏品堆叠趋势、空隙、凸出物品和已知线索命中的位置。

玩家卡片内容：
- 头像 / 名称。
- 当前金币（自己的实时显示，NPC 的只在被"窥探"后显示）。
- 每轮道具记录（自己的显示具体道具，NPC 默认显示"?"；若后续有公开道具或探查结果，再显示具体名称）。
- 每轮排名记录（第 1 轮结束后开始累计；当前轮未结算时显示"-"）。
- 只有上一轮触发"提前拿下"或进入终局回顾时，才展示具体出价金额。
- 最终轮结束后才显示所有玩家每轮的完整出价。

中间信息区：
- 公共信息卡片：地图线索、场所资料、公开提示、所有人都能看到的 NPC 行为。
- 个人信息卡片：个人道具线索、窥探/窃听结果、只对自己可见的估价依据。
- 卡片按获得时间从上到下排列；信息过多时中间区域可以滚动。
- 重要线索可以高亮边框，并同步高亮右侧仓库中的命中格子。

右侧仓库与估价：
- 仓库网格只展示仓库结构和藏品轮廓，不承载出价按钮。
- 最低估价：实时基于线索计算更新。
- 已知藏品数量、未知藏品范围、线索命中的格子位置在仓库旁显示。
- 右上角提供"藏品百科"入口，打开后显示筛选和藏品卡片列表。

底部操作条：
- 道具按钮打开本局携带道具槽位。
- 主按钮根据状态显示"出价"、"已出价：金额"、"确认加拍"等；点击"出价"打开出价弹层。
- 放弃按钮用于本轮不出价；提交后主按钮锁定，放弃按钮隐藏或置灰。

交互：
- 道具使用即时生效，线索和估价即时刷新。
- 点击道具槽位直接使用；库存为 0、已使用或本轮已用过道具时置灰。
- 需要目标的道具（如"窥探"、"窃听"）点击后进入目标选择状态，再点击某个 NPC 生效。
- 点击"藏品百科"打开百科弹层，关闭后回到出价界面，不中断倒计时。
- 点击仓库内已知或被线索命中的格子，可在中间信息区定位到对应线索。
- 倒计时结束自动提交（有输入且合法）或视为放弃。
- 确认出价后底部主按钮显示"已出价：金额"；倒计时结束前可再次打开出价弹层修改金额。

### 出价弹层

出价不直接在主界面输入，点击底部主按钮后弹出确认面板，减少误触。

```
┌────────────────────────────────────┐
│  本轮出价                           │
│                                    │
│  当前金币：9,750,000                │
│  当前最低估价：1,200,000            │
│  可出价上限：9,750,000              │
│                                    │
│  出价金额：[____________]           │
│  [最低估价] [一半金币] [全部金币]    │
│                                    │
│  [取消]              [确认出价]      │
└────────────────────────────────────┘
```

交互：
- 输入金额必须大于等于 1，且不能超过当前金币。
- 快捷按钮只填入金额，不自动提交。
- 点击"确认出价"后关闭弹层，底部主按钮变为"已出价：金额"。
- 已提交后允许在倒计时结束前修改出价；再次点击"已出价：金额"重新打开弹层。
- 倒计时结束时使用最后一次确认的出价；没有确认过则视为放弃。

### 藏品百科弹层

藏品百科用于辅助玩家判断线索价值，不直接暴露本轮真实仓库结果。

```
┌────────────────────────────────────────────────────────────────────┐
│ 藏品百科                                                    [关闭] │

└────────────────────┴───────────────────────────────┴─────────────┘
```

交互：
- 示例藏品来自 `shared/config/collectibles.json`。
- 点击品质、类型、轮廓筛选项后，藏品列表和平均价实时刷新。
- 藏品卡片展示名称、类型、稀有度、轮廓、样图和参考价格。
- 百科中的价格是全局参考信息，不等于本轮仓库真实总价。
- 打开百科时背景对局界面压暗，但右侧仓库仍可见，用于对照轮廓和线索。
- 第一阶段可以先做只读百科，不做复杂搜索。

## 4. 出价结果与加拍

所有玩家提交或倒计时结束后，先进入出价结果判定界面。这个界面只负责展示本轮出价结果、提前拿下状态、平局加拍入口或流拍结果。

```
┌──────────────────────┬──────────────────────────────────┐
│  本轮结果             │  第 2 轮 出价结果                 │
│                      │                                  │
│  获胜者               │  你：第 1 名                     │
│  玩家 1 (你)          │  NPC-A：第 4 名                  │
│                      │  NPC-B：第 3 名                  │
│  状态                 │  NPC-C：第 2 名                  │
│  普通成交             │                                  │
│                      │  [查看仓库揭秘]                  │
│  下一步               │                                  │
│  仓库揭秘             │                                  │
└──────────────────────┴──────────────────────────────────┘
```

普通成交：
- 无人触发"提前拿下"时，只展示出价排名，不展示具体金额。
- 玩家自己的排名高亮显示。
- 点击"查看仓库揭秘"进入成交揭示。

提前拿下：
- 顶部显示"提前拿下！"。
- 展示所有玩家本轮具体出价金额。
- 仍然点击"查看仓库揭秘"进入成交揭示。

并列最高价：
- 如果出现并列最高价，界面进入"加拍等待"状态。
- 只有并列最高者显示加拍输入框，其余玩家只看到等待提示和当前加拍轮次。
- 加拍仍然是暗拍，输入金额必须高于自己上一轮最高出价，且不能超过当前金币。
- 如果玩家不在并列最高者中，则只能旁观加拍结果。
- 如果只有 NPC 之间并列最高，第一阶段可由本地逻辑自动完成 NPC 加拍并快速展示结果。
- 加拍阶段不允许再使用道具，道具只在本轮初始出价阶段使用。
- 每次加拍后重新进入结果判定；最多加拍 10 次。
- 10 次后仍未决出胜者时，本轮流拍。

```
┌──────────────────────┬──────────────────────────────────┐
│  平局加拍             │  第 2 轮 加拍 1/10                │
│                      │                                  │
│  并列最高             │  参与者：你、NPC-C                │
│  你 / NPC-C           │                                  │
│                      │  上次你的出价：1,200,000         │
│  本轮要求             │  加拍：[__________]              │
│  必须高于上次出价      │                                  │
│                      │  [确认加拍] [放弃加拍]            │
└──────────────────────┴──────────────────────────────────┘
```

流拍：
- 所有玩家都放弃，或加拍 10 次仍然平局时，本轮仓库作废。
- 不扣除本轮出价金币。
- 已扣除的入场费和已消耗道具不返还。
- 点击"进入下一轮"继续；第 5 轮流拍后进入终局结算。

## 5. 成交揭示

成交揭示沿用场所背景，但整体压暗，突出结果信息。左侧显示获胜者和收益，右侧显示战利品仓库逐件揭示，中间保留场景空间用于增强沉浸感。

```
┌──────────────────────┬──────────────────────────────────┐
│  获胜者               │  第 2 轮 仓库揭秘                 │
│                      │                                  │
│  玩家 1 (你)          │  ┌────┐                         │
│  ┌────┐              │  │    │ 明代青花瓷瓶              │
│  │头像│              │  │    │ R 级 · 瓷器               │
│  └────┘              │  └────┘ 真实价值：450,000        │
│                      │                                  │
│  拿下价格             │  ┌──┐                           │
│  1,200,000           │  │  │ 清乾隆粉彩碗               │
│                      │  └──┘ SR 级 · 瓷器               │
│  本轮利润             │  真实价值：800,000              │
│  +1,250,000          │                                  │
│                      │  ... 自动逐件加载中 ...          │
│  累计利润             │                                  │
│  +2,500,000          │                                  │
│                      │                                  │
│  [提前拿下！]         │                                  │
│                      │  [下一件] [加速 x2] [跳过动画]    │
│                      │                                  │
└──────────────────────┴──────────────────────────────────┘
```

布局：
- 背景层：当前场所场景压暗，减少出价阶段的信息噪声。
- 左侧 30%：对局结果标题、获胜者头像、拿下价格、战利品总价、本轮利润和累计利润。
- 中间 35%：保留场景背景和点击继续提示，不堆叠大量 UI。
- 右侧 35%：战利品仓库，藏品从轮廓状态逐件变为真实藏品状态。
- 如果是提前拿下，左侧顶部显示"提前拿下！"标记。

揭示节奏：
- 藏品从左到右、从上到下依次加载（与仓库中位置一致）。
- 每件出现时带短暂动画（如 0.5s 淡入），然后停留 1~2 秒。
- 件与件之间有 0.3s 间隔。
- 每加载一件藏品，左侧利润数字实时跳动更新。
- 默认自动逐件播放。
- 提供"下一件"、"加速"、"跳过动画"按钮，避免 30~60 件藏品揭示时间过长。
- 全部加载完毕后停留 3 秒，显示"下一轮"或"终局结算"按钮；如果玩家不操作，可自动过渡。

揭示时的视觉：
- 格子中的藏品色块从暗色轮廓翻转/展开为真实藏品图，显示名称、稀有度、类型、真实价值。
- 刚从隐藏状态变为揭示状态的藏品有高亮闪烁效果。
- 利润数字跳动动画。
- 正利润使用绿色数字，负利润使用红色数字。

状态提示：
- **提前拿下**：左侧保留"提前拿下！"标记，具体出价金额已在结果页展示。
- **普通成交**：左侧显示获胜者、拿下价格和本轮收益，不重复展示所有出价。
- **玩家落败**：左侧显示获胜者信息，玩家自己的排名已在结果页展示。
- **流拍**：不进入成交揭示页，直接从结果页进入下一轮或终局结算。

## 6. 后续轮次（第 2~5 轮）

同上。第 1 轮无历史排名，第 2 轮起左侧玩家栏展示已完成轮次的道具和排名。

轮次开始：
- 进入新一轮前短暂显示"第 N 轮开始"提示。
- 立即扣除本轮入场费，并在中间信息流添加一条系统卡片。
- 生成新仓库和本轮公共线索，右侧战利品仓库重置为未知状态。
- 道具使用限制重置为"本轮可使用 1 个"。

第 5 轮：
- 第 5 轮结果和成交揭示完成后进入终局结算。
- 第 5 轮可展示更完整的压力反馈，例如倒计时颜色变化、最终轮标记。

## 7. 终局结算

- 左侧：只展示本次拿下者信息和结算数字，不展示 4 个玩家排行榜。
- 中间：尽量保留场景背景和"点击这里继续"提示，不堆叠复杂 UI。
- 右侧：战利品仓库占据屏幕右侧整列，宽度约 32% 到 36%，从顶部标题栏延伸到底部。

- 右侧顶部保留"战利品"标题和"藏品百科"入口。
- 点击某件藏品可查看名称、稀有度、类型、真实价值和卖出价。
- 点击"出价回顾"打开弹层，查看所有玩家每轮完整出价。
- 点击"进入个人仓库"查看自己已获得且未出售的藏品；如果本次拿下者不是玩家自己，该按钮仍进入玩家自己的仓库。
- 如果触发亏损均摊，在结算页增加"亏损均摊"提示行，展示均摊前净收益、均摊金额和均摊后净收益。
- 点击"返回大厅"回到大厅界面。

## 8. 个人仓库与卖出

个人仓库用于承接每局获得但未出售的藏品，也用于对局结束后的手动卖出。

```

│  个人仓库                                      金币 985万  │
│                                                            │
```

交互：

- 点击藏品详情可查看来源轮次、真实价值、卖出价、类型、稀有度和尺寸。
- 点击"卖出选中"后弹出二次确认，确认后增加金币并移除对应藏品。

- 未卖出的藏品保留在个人仓库，参与下局开局资产计算。

## 状态流转

```
主菜单
  ├─→ 道具配置 ─→ 主菜单
  └─→ 大厅
        ├─→ 开局确认 / 调整道具
        └─→ 出价阶段（第1轮）
              └─→ 出价结果
                    ├─→ 普通成交 / 提前拿下 ─→ 成交揭示
                    │       ├─→ 出价阶段（下一轮）  ← 循环 5 轮
                    │       └─→ 终局结算
                    ├─→ 并列最高 ─→ 加拍阶段 ─→ 出价结果
                    └─→ 流拍

                            └─→ 终局结算
                                  ├─→ 个人仓库 / 卖出
                                  └─→ 大厅
```

## 第一阶段实现优先级

第一阶段目标是尽快验证核心循环，不需要一次性做完所有视觉细节。

必须先做：
- 大厅、开局确认、出价主界面、出价弹层、出价结果、成交揭示、终局结算。
- 左侧玩家轮次记录、中间信息流、右侧仓库与最低估价。
- 道具面板只需要支持 5 个槽位和每轮最多使用 1 个。
- 藏品百科先做只读筛选，不做复杂搜索和收藏管理。

可以延后：
- 场所背景的多套变体。
- 藏品真实图片和高级翻转特效。
- 个人仓库的复杂排序、批量筛选和展示动画。
- 联机状态、聊天、好友、付费和角色技能。

## 待细化

- 场所背景图和 UI 半透明面板的可读性：需要在 Unity 中验证不同亮度背景下的遮罩强度。
- 仓库网格的视觉比例：需要在 Unity 中验证缩略总览、滚动区域和藏品格子尺寸的实际可读性。
- NPC 出价动画：出价阶段 NPC 的"已出价"状态何时展示（建议所有玩家提交后或倒计时结束后统一展示）。
- 藏品轮廓剪影：MVP 阶段轮廓从 `size` 自动派生（矩形色块加虚线边框表示尚未揭示的藏品）。
- 成交揭示节奏：0.5s 淡入 + 1.5s 停留 + 0.3s 间隔 为默认值，可配置。
